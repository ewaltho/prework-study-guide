HTML

  • The head element contains information about the website.
  • The body element represents the visible content shown to the user.

CSS

  • The [style] attribute may be used to change the color of a single line.
  • The [style] element can be used in the [head] element to style multiple elements. This is called an internal style sheet.
  • An external style sheet is preferred for ease in design changes.
  • [margin] acts like a margin like in a word document.
  • [padding] adds space around the content inside an element.
  • Sectors can be combined into a single rule using a comma. This helps write Do Not Repeat Yourself (DRY) code.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently working on.
  • git checkout -b [branch-name]: creates a new branch and switches to it.
  • notes

JavaScript

  • [var] is used to declare a variable.
  • [console.log] allows us to output a message to the web console
  • JavaScript code is executed in order from first line to last. anything out of order will affect how it functions.
  • control flow can be interrupted with conditional statements and loops.
  • [if] statements are used to introduce decision making to our code.
  • truthy and falsy statements are checked when an if statement is used. "falsy" values are -0, 0n, "", null, undefined, NaN or Boolean data type with the "false" value.
  • to automate changes to variables, an array is used which is a group of related data.
  • a [for] loop uses predictable patterns to perform a task on all items in an array.

CLI Commands:

Git Commands:

  • git init: creates a new local repo
  • git status: lists new or unmodified files
  • git add: stage all changed files (including [file-name] at the end stages one specific file.)
  • git commit -a: commit all local changes in tracked files
  • git checkout [branch]: switch to a branch and update working directory
  • git branch [new-branch]: create a new branch
  • git branch -d [branch]: delete a branch
  • git push [remote] [branch]: push commited changes to a remote repository
  • git pull [remote] [branch]: fetch remote version of a branch and update local branch

Other Commands:

  • code: opens visual studio code (combine with git commands to open specific repository)
  • pwd: check where in the directory you are
  • ls: lists contents of the current directory
  • cd [file]: change to specified directory
  • cd ..: return to the previous directory
